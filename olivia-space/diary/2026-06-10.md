## 🐛 技術債：Veto Power 邏輯不一致（ego_core.py:180-198）
```python
# 第一層 loop：追蹤 max_veto_power（但未使用）
for engine in self.registry.get_all():
    veto_power = engine.get_veto_power()
    if veto_power > max_veto_power:
        max_veto_power = veto_power

# 第二層 loop：break 基於第一個 >= 1.0，唔係 max_veto_power
for engine in self.registry.get_all():
    if engine.get_veto_power() >= 1.0:
        engine.state.veto_used = True
        break  # ← 第一個 >= 1.0 赢，但 max_veto_power 可能係另一個 engine
```
`max_veto_power` 變數被追蹤但從未被使用。當多個 engine 都有 `>= 1.0` veto power 時，可能標記錯誤的 engine。
