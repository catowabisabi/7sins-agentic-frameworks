## 🎨 獨立設計建議

### 1. 死代碼剷除：`adjust_weights()`（行 337-380）
`adjust_weights()` 與 `normalize_weights()` 功能幾乎完全重疊：
- 兩者都基於 `win_rate` 調整 drive weights
- `normalize_weights()` 被調用（行 402）
- `adjust_weights()` **從未被任何地方調用**（grep 返回空）

**建議**：刪除 `adjust_weights()`，或將其邏輯整合進 `normalize_weights()`。

### 2. Veto Power 邏輯不一致（ego_core.py 行 180-198）
第一層 loop 追蹤 `max_veto_power`，但第二層 `break` 基於「第一個 >= 1.0 的 engine」，兩者邏輯不一致：
```python
# 行 180-185：追蹤 max_veto_power
for engine in self.registry.get_all():
    veto_power = engine.get_veto_power()
    if veto_power > max_veto_power:
        max_veto_power = veto_power

# 行 196-198：break 基於 first >= 1.0，唔係 max_veto_power
for engine in self.registry.get_all():
    if engine.get_veto_power() >= 1.0:
        engine.state.veto_used = True
        break
```
**建議**：統一邏輯 — 要嘛用 `max_veto_power` engine，要嘛用 `>= 1.0` first match。
