### Veto Power 邏輯不一致（ego_core.py:180-198）
第一層 loop 追蹤 `max_veto_power`，但第二層 `break` 只係基於「第一個 >= 1.0 的 engine」：
```python
# 行 180-185：追蹤 max_veto_power
for engine in self.registry.get_all():
    veto_power = engine.get_veto_power()
    if veto_power > max_veto_power:
        max_veto_power = veto_power

# 行 196-198：break 基於 first >= 1.0，唔係 max_veto_power
for engine in self.registry.get_all():
    if engine.get_veto_power() >= 1.0:
        engine.state.veto_used = True
        break
```
如果多個 engine 同時有 `>= 1.0` veto power，會取第一個（iterator order），唔係 `max_veto_power` 嗰個。
