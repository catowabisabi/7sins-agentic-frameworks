## 🎨 獨立設計建議

### 1. Veto Power 機制需要更清晰嘅觸發條件
**問題**: `veto_power >= 1.0` 係 hardcoded threshold，但 `get_veto_power()` 返回 `weight * (1.0 if veto_used else 0.5)`。呢個邏輯：
- 如果 `veto_used = True`，返回 `weight * 1.0`（但 weight 通常 < 1.0，所以永遠 < 1.0）
- 如果 `veto_used = False`，返回 `weight * 0.5`（更細）
- **結論**: `veto_power >= 1.0` 幾乎不可能觸發！呢個係邏輯 bug，定係設計缺陷？

**建議**: 
- 重新思考 veto power 的 scaling 機制
- 或者將 threshold 改為 `veto_power >= weight`（即已使用的 drive）

### 2. normalize_weights() vs adjust_weights() 需要文件說明
**問題**: #29 將 `adjust_weights()` 改為 `normalize_weights()`，但冇解釋兩者嘅分別同適用場景。

**建議**: 在 `drive_engine.py` 加上 docstring 說明：
- `adjust_weights()`: 根據 win_rate 調整權重（歷史反饋）
- `normalize_weights()`: 確保所有權重總和為 1.0（狀態一致性）

### 3. ReflectionAgent Singleton 需要測試覆蓋
**問題**: #30 consolidate 咗 dual ReflectionAgent，但 shared instance 之後：
- 多線程並發安全性？
- veto_used 狀態係咪需要在 reset 之後正確重置？

**建議**: 增加 unit test 驗證 shared instance 行為。

### 4. Seven Sins 框架自身應用
user-intention.md 提到 LLM 適合「Human-in-the-loop」，但 7Sins 嘅 veto power 機制正正係一種「Machine-in-the-loop」— 機器有否决權。這係一個有趣的矛盾：
- 7Sins 系統讓 drive agents 有否决權
- 但 user-intention 說 LLM 不適合完全自主決策

**建議**: 在 `seven_sins.py` 或 `ego_core.py` 加入註解，說明呢個系統係「Shared Agency」模式，而非完全自主。
