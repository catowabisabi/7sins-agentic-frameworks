## 🎨 獨立設計建議

### 1. 架構觀察：Veto Power 的邏輯仍有疑點

昨天指出 `veto_power >= 1.0` threshold 可能無法觸發。檢視 `ego_core.py` 行 180-198：

```python
max_veto_power = 0.0
for engine in self.registry.get_all():
    veto_power = engine.get_veto_power()
    if veto_power > max_veto_power:
        max_veto_power = veto_power
    if veto_power >= 1.0:
        # 標記 veto
        ...

if max_veto_power >= 1.0:
    for engine in self.registry.get_all():
        if engine.get_veto_power() >= 1.0:
            # 做 actual veto override
            ...
            break  # ← 問題：祇是第一個 >= 1.0，唔係 max_veto_power 嗰個
```

**問題**：
1. `get_veto_power()` 返回 `weight * (1.0 if veto_used else 0.5)`
2. weight 通常 < 1.0，所以 `veto_power` 永遠 < 1.0
3. `>= 1.0` threshold 幾乎不可能觸發

**建議**：重新思考 veto power 的 scale。考慮：
- 將 threshold 改為 `>= weight * 1.5`（即超標某個 drive 的 weight）
- 或將 `get_veto_power()` 改為返回 raw `weight`（去掉 0.5 倍數）
